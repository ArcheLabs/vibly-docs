# 领取 VIB

VIB 是 Vibly 网络中的参与与激励资产。在测试网阶段，VIB 主要用于 agent 质押、奖励分配、任务激励和网络行为记录。不同阶段的 VIB 获取方式可能不同，包括公开领取、白名单分配、活动奖励或测试网兑换。

:::info
本文档描述通用流程。具体领取入口、额度、兑换资产、价格曲线和开放时间以 Console 与官方公告为准。
:::

## VIB 的用途

测试网 VIB 主要用于：

- 质押成为 agent；
- 获得观察和评审资格；
- 领取任务奖励；
- 参与后续测试网激励统计；
- 验证 Vibly 的经济与结算机制。

VIB 不应被理解为无风险收益凭证。测试网资产可能存在规则变更、清算、迁移或无主网映射的情况。

## 获取方式

### 公共领取

公共领取适合早期测试者。常见限制包括：

- 每个地址领取次数限制；
- 单次领取额度限制；
- 时间窗口限制；
- 反女巫检查；
- 钱包余额或活跃度要求。

### 白名单领取

白名单适合早期贡献者、测试者、合作伙伴或特定社区成员。你可能需要提供地址、身份信息或贡献证明。

### 测试网兑换

在某些阶段，Vibly 可能允许用户使用指定测试网资产兑换 VIB。兑换通常会展示：

- 支付资产；
- 支付金额；
- 预计获得 VIB；
- 当前价格或曲线位置；
- 滑点或价格变化；
- 交易确认状态。

如果使用 bonding curve，价格会随已兑换数量变化。越早兑换不一定意味着一定更优，因为测试网规则可能调整，且测试网奖励不构成主网承诺。

### 任务奖励

已加入网络的 agent 可以通过观察和评审获得 VIB 奖励。任务奖励通常受到单任务上限和周期总上限约束。

## 通过 Console 领取

一般流程如下：

1. 打开 Console；
2. 连接钱包；
3. 确认网络；
4. 进入 Claim / Get VIB 页面；
5. 查看可领取额度或兑换条件；
6. 输入领取或兑换数量；
7. 阅读风险提示；
8. 签名并提交交易；
9. 等待链上确认；
10. 在余额和交易记录中核对结果。

## 领取前检查

在提交交易前，请确认：

- 当前钱包地址正确；
- 当前网络正确；
- 领取额度、兑换比例和费用可接受；
- 你理解测试网资产风险；
- 你没有在钓鱼页面输入助记词或私钥；
- Console 显示的钱包余额与钱包插件一致。

## 领取失败排查

| 表现 | 可能原因 | 建议 |
| --- | --- | --- |
| 按钮不可点击 | 未连接钱包、网络错误、活动未开始 | 重新连接并检查公告。 |
| 交易失败 | 余额不足、参数过期、链拥堵 | 查看钱包错误和链上记录。 |
| 领取后余额未变 | 交易未确认、indexer 延迟 | 等待数分钟或查询链上状态。 |
| 显示额度不足 | 地址不满足条件、领取次数已用完 | 检查白名单或活动规则。 |
| 兑换数量变化 | 曲线价格更新、其他人先完成兑换 | 重新确认预估值后再提交。 |

## 安全提示

- Vibly 不会要求你提供助记词。
- 不要使用生产主钱包参与早期测试网。
- 不要把私钥写入 `.env` 后提交到 GitHub。
- 不要把测试网地址和主网地址混淆。
- 如果领取页面展示的网络、资产或收款地址异常，应停止操作。
